db module

class db.DatabaseManager[исходный код]

Базовые классы: object

Менеджер базы данных для работы с CSV и JSON файлами.

Обеспечивает импорт/экспорт данных, а также создание необходимых файлов для хранения информации о клиентах, товарах и заказах.

create_files_if_not_exist()[исходный код]

Создает необходимые файлы CSV, если они не существуют.

Проверяет наличие файлов customers.csv, products.csv, orders.csv и order_items.csv. Если какой-либо файл отсутствует, вызывает метод save_data соответствующей модели для создания файла с заголовками колонок.

Заметки

Метод должен вызываться при инициализации приложения для обеспечения наличия всех необходимых файлов данных.

export_to_csv(data, filename, fieldnames)[исходный код]

Экспортирует данные в CSV файл.

Параметры:

  • data (list of dict) – Список словарей с данными для экспорта. Каждый словарь представляет одну строку данных, где ключи соответствуют названиям колонок.

  • filename (str) – Имя файла для сохранения (например, „export.csv“).

  • fieldnames (list of str) – Список названий колонок в правильном порядке.

Результат:

True если экспорт выполнен успешно, False при возникновении ошибки.

Тип результата:

bool

Примеры

>>> data = [{'name': 'John', 'age': 25}, {'name': 'Alice', 'age': 30}]
>>> fieldnames = ['name', 'age']
>>> export_to_csv(data, 'people.csv', fieldnames)
True
export_to_json(data, filename)[исходный код]

Экспортирует данные в JSON файл.

Параметры:

  • data (list or dict) – Данные для экспорта. Могут быть списком словарей или словарем.

  • filename (str) – Имя файла для сохранения (например, „export.json“).

Результат:

True если экспорт выполнен успешно, False при возникновении ошибки.

Тип результата:

bool

Заметки

Файл сохраняется с отступами (indent=4) и поддержкой Unicode (ensure_ascii=False).

import_from_csv(filename, model)[исходный код]

Импортирует данные из CSV файла в указанную модель данных.

Параметры:

  • filename (str) – Имя файла для импорта (например, „import.csv“).

  • model (BaseModel) – Модель данных для добавления элементов. Должна иметь метод add_item().

Результат:

Количество успешно импортированных записей. Возвращает -1 при ошибке.

Тип результата:

int

Исключение:

  • FileNotFoundError – Если указанный файл не существует.

  • PermissionError – Если нет прав на чтение файла.

Заметки

Метод использует метод add_item модели для валидации и добавления каждого элемента данных.

import_from_json(filename, model)[исходный код]

Импортирует данные из JSON файла в указанную модель данных.

Параметры:

  • filename (str) – Имя файла для импорта (например, „import.json“).

  • model (BaseModel) – Модель данных для добавления элементов. Должна иметь метод add_item().

Результат:

Количество успешно импортированных записей. Возвращает -1 при ошибке.

Тип результата:

int

Исключение:

  • FileNotFoundError – Если указанный файл не существует.

  • JSONDecodeError – Если файл содержит некорректный JSON.

Заметки

Метод ожидает, что JSON файл содержит массив объектов или объект, который можно итерировать.
